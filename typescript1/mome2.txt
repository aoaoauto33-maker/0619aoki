・Interface

・リテラル型
変数が取りうる値を特定の具体的な値に限定する型(const age = 18;)
リテラル...変数に入れる実際の値(18)
これは数値の型ではなく、18という値の入った型、:numberと書けば数値型になる


・ユニオン型
独自で作れる型(スゲー)
| で区切る　どっちかが入るよーって意味
文字列も数値も入れられるようになる超便利


・リテラル型との組み合わせ
リテラル型を | で複数組み合わせる

・型エイリアスとinterfaceは何が違うの？
interfaceの方がオブジェクトの形状に明確な定義があって、強制力もあり安全性が高い


interface Person {
    name: string,
    age: number;
    speak(word: string): void;
    オブジェクトの中にある関数はメソッド
}





・インデックスシグニチャ
name: string のnameの部分は他の名前が入る可能性があるため、型だけ設定しておくやつ







・インターフェイスと型エリアスの違い
インターフェイスはオブジェクト特化、型エリアスは汎用的
インターフェイスはクラスと連携プレー、型エリアスは無理
